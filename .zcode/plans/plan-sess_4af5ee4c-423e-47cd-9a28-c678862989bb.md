## Today React 아키텍처 재구성 계획

**원칙**: 1:1 동작 보존 (화면/목업/인터랙션 동일). `domain/task.ts`(순수 로직, 테스트 검증됨)과 `components/ui/`(shadcn)는 건드리지 않음. PRD 기술 원칙에 따라 Repository/데이터 추상화는 만들지 않음.

---

### 현재 문제 → 해결

| 문제 | 해결 |
|---|---|
| 만능 스토어(258줄, 상태+액션+목업+테마+localStorage 뒤섞임) | 관심사별 5개 슬라이스로 분리 |
| 도메인 타입 중복(`Task`가 store에, `TaskStatus`가 domain에) | `domain/types.ts`로 엔티티 타입 통합 |
| 사이드이펙트가 컴포넌트에 산재(테마/창/오늘날짜) | `lifecycle/providers` + `lifecycle/hooks`로 중앙화 |
| `tasks→records` 변환 3곳 복붙(App/HistoryView/MonthlyCalendar) | `domain/derive.ts` + 파생 훅으로 단일화 |
| `App.tsx`가 7가지 역할(라우팅/헤더/푸터/서랍/창/스트릭/옵션) 과부하 | `app/` 폴더로 컴포넌트 분할 |
| 전체 구독으로 과잉 리렌더 | 원자적 selector `useStore((s) => s.x)` 적용 |
| 스토어가 DOM 결합(`applyTheme`이 document 조작) | `lib/theme.ts` + `ThemeProvider`로 분리 |

### 핵심 설계 결정

**1. 스토어 분리 + 원자적 selector**
```ts
// AS-IS: 전체 구독 (과잉 리렌더)
const { tasks, friends } = useTodoStore();

// TO-BE: 원자적 selector
const tasks = useTaskStore((s) => s.tasks);
const restWeekdays = useSettingsStore((s) => s.restWeekdays);
```

**2. 크로스 스토어 의존성 제거** — 액션이 필요한 값을 파라미터로 받음
```ts
// useTaskStore
addTask: ({ title, tag, date }) => ...        // selectedDate를 useUiStore에서 안 읽음
resolveRollover: (id, decision, currentDate) => ...
// useDiaryStore
addDiary: (content, date) => ...               // getTodayStr() 내부 호출 제거
```
→ 각 스토어가 다른 스토어를 모름, 테스트 용이.

**3. 파생 데이터 단일화** — 3곳 복붙 제거
```ts
// domain/derive.ts
export function tasksToCommitmentRecords(tasks): CommitmentRecord[] { ... }
// hooks/useCurrentStreak.ts
export function useCurrentStreak(): number {
  const records = useCommitmentRecords();
  const restWeekdays = useSettingsStore((s) => s.restWeekdays);
  const today = useToday();
  return useMemo(() => commitmentStreak(records, today, restWeekdays), [...]);
}
```

**4. 생명주기 Provider 분리** — 사이드이펙트를 컴포넌트에서 제거
- `LocaleProvider`: `todayStr`을 Context로 1회 계산 (App/History/MonthlyCalendar의 `useMemo` 중복 제거)
- `ThemeProvider`: `theme` 변경 시 `applyTheme` 호출 (DOM 조작은 `lib/theme.ts`로)
- `WindowPersistence`: Tauri 창 크기 저장/복원 (App.tsx의 40줄 useEffect 이동)

**5. App.tsx 분할** — 역할별 컴포넌트
```tsx
// App.tsx (15줄)
export default function App() {
  return (
    <AppProviders>
      <AppShell />
    </AppProviders>
  );
}
// AppShell = 햄버버버튼 + main(AppHeader + ViewRouter) + AppNavigation + OptionDrawer
```
