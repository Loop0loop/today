## 홈 화면 할 일 입력 UX 재설계

### 결정된 방향
- **플로팅 입력 바** (하단 네비 캡슐 바 위, 고정)
- **태그**: 입력 후 선택 (기본값 없음 → Enter 시 자동 분류, 필요할 때만 태그 지정)
- **약속 입력**: 입력 바 안 [약속/일반] 토글 (약속 모드 시 빈 슬롯 자동 배정)

### 변경 범위

**1. `src/components/TodayView.tsx`** (핵심)

제거:
- 기존 인라인 입력 폼 (216–267줄: Input + 드롭다운 + 항상 노출되는 태그 칩)
- 빈 약속 슬롯의 `+ 슬롯 N 지정` 인라인 입력 (340–374줄) → 빈 슬롯은 단순 placeholder 문구만 표시 ("슬롯 N 비움")

추가 — 하단 플로팅 입력 바 (TodayView 내부, `fixed bottom-20`로 네비 위):
- **모드 토글**: `[⚡ 일반] [🎯 약속]` 세그먼트. 기본 "일반"
  - 약속 모드: `nextCommitmentSlot` 헬퍼로 빈 슬롯 자동 감지. 3개 다 찼으면 토글 비활성화 + "오늘 약속은 모두 정했어요" 안내
- **입력 필드**: 넓은 rounded 바. 모드별 맥락형 placeholder
  - 일반: "오늘 할 일을 적어보세요"
  - 약속: "오늘의 핵심 약속 하나 (최대 3개)"
- **예시 칩** (일반 모드 + 입력창 비어있을 때만): `💡 물 마시기` `📖 책 20p` `🏃 10분 산책` `💬 감사인사` — 클릭 시 입력창에 채움 (자동 제출 아님). "뭘 적어야 할지 모르겠다" 직접 해소
- **태그 토글**: 입력창 우측 `#` 버튼 → 클릭 시 입력 바 내에 태그 칩 4개 펼침/접힘. 선택하지 않아도 Enter 가능 (기본값 없음 = 자동 분류)
- **추가 버튼**: Enter 또는 클릭으로 제출. 제출 후 모드/태그 초기화

**2. `src/App.tsx`**
- `main`의 `pb-32`를 `pb-44`로 증가 — 플로팅 입력 바 높이만큼 스크롤 영역 하단 여백 확보 (마지막 할 일이 가려지지 않게)

### 변경하지 않는 것
- `useTodoStore.ts` — 기존 `addTask`, `addCommitment`, `nextCommitmentSlot` 그대로 재사용
- 약속/할 일 표시 로직, 롤오버 섹션, 월간 캘린더 — 그대로 유지
- 태그 값(BASS/DEV/etc/PLAY) 체계와 색상 — 유지

### 구현 순서
1. TodayView에서 기존 입력 UI 제거 + 빈 슬롯 placeholder 단순화
2. 플로팅 입력 바 JSX + 상태(`inputMode`, `showTags`, `newTodoText`, `selectedTag`) 추가
3. 모드별 제출 로직 (약속=자동 슬롯 배정, 일반=태그와 함께)
4. 예시 칩 데이터 + 클릭 입력 로직
5. App.tsx 하단 padding 조정
6. `pnpm check` 로 타입/테스트 검증

### 검증 기준
- 빈 할 일 목록에서도 "뭘 적어야 할지" 힌트가 보인다 (예시 칩)
- 입력 바가 스크롤과 무관하게 항상 네비 위에 보인다
- 태그를 고르지 않고 Enter 쳐도 할 일이 추가된다
- 약속 3개가 다 찼을 때 약속 모드가 막히고 안내가 나온다
- 기존 약속/할 일/롤오버 동작이 깨지지 않는다
